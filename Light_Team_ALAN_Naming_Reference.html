LIGHT Team 🐦 File Naming Reference

Quick Reference, ALAN “W” Study, Impact Site, Baseline Year 2026

Weber State University Dr. John F. Cavitt Posted: April 2026
Critical: raw files are NEVER renamed at ingest
Device-generated files must be ingested exactly as produced. This convention applies to derivative and processed outputs only (detection results, summaries, exports). Renaming raw files breaks SHA-256 checksum verification.
Keep as-is:  SM4_20260412_023012.wav  |  20260412_023012.WAV  |  IMG1.FIT  |  DS000001.mp4
If a naming error occurs after ingest: do not delete. Move to _to_rename/, create the correctly named version in the correct subfolder, and document both in the Field Log (SOP 13).
All data exports must use a lossless file format
Tabular data (sensor readings, measurements): export as .csv, never XLSX.
Scientific images (dark frames, all-sky frames): export as .fit (FITS) where available, PNG is the required fallback if FITS is not supported by the device.
Screen captures and field photographs: export as .png, never JPG or BMP.
Audio: keep device-generated format, .wav or .WAV as produced. Never convert or re-encode.
Video: keep device-generated format, .mp4 as produced. Never re-encode.

JPG compression permanently destroys pixel data and is not acceptable for any scientific export in this project. If your device only offers JPG, contact the Data team before exporting.
Naming convention questions: contact the Data team
If you are unsure how to name a file, which subfolder to use, or whether your device export matches the convention, do not guess. Contact the Data and QA Coordinator before ingesting. It is easier to name files correctly at ingest than to rename them after the manifest has been generated.

Filename handling notes

Time zone rule: all derivative filenames use UTC
MDT = UTC −6 hours. Evening sessions roll to the next calendar day in UTC.
Data collected 8 pm April 12 → filename date is 20260413, not 20260412.

2b. Device export notes

Three sensors produce device-generated filenames
AudioMoth: generates YYYYMMDD_hhmmss.WAV. Never renamed. Attribution is by subfolder only: AM/N/ or AM/S/.

Spectrometer: generates three file types per session: (1) measurement CSV 2026_I_SPEC_01_NN_CCTK.csv, (2) spectral distribution image, device exports as JPG but must be saved as PNG, (3) screen capture PNG exported manually. Set memory title to 2026_I_SPEC_01 before each season (update the year prefix each season, e.g., 2027_I_SPEC_01). Raw files are never renamed. Spectral distribution images go in SPEC/raw/, dark frames in SPEC/dark/, screen captures in SPEC/screencap/.

HOBO: HOBOware generates its own export filename. Place in the correct numbered subfolder (HOBO/01/ through HOBO/06/) without renaming.
